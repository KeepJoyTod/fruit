这是一份直接可用的微信小程序技术实现方案，围绕“街边水果摊上架展示”场景，用 **JavaScript + 微信云开发** 全栈实现，让你能快速启动开发。

---

## 一、技术栈推荐

| 层 | 技术选型 | 说明 |
|---|---------|------|
| 前端框架 | 微信原生框架（WXML + WXSS + JS） | 无需额外学习成本，直接调用摄像头、地图等能力 |
| 类型系统 | TypeScript（可选） | 提高代码可维护性，微信开发者工具内置模板 |
| 后端/云服务 | **微信云开发** | 免运维，自带数据库、存储、云函数 |
| 云函数运行环境 | Node.js (JavaScript) | 编写后端业务逻辑 |
| 数据库 | 云数据库（文档型，类MongoDB） | 操作语法在 JS 中直接使用 |
| 图片存储 | 云存储 | 上传水果照片，自动返回 fileID |
| 地图服务 | 微信内置地图组件 + 腾讯地图API | 用于摊位定位与导航 |
| 图像识别（可选） | 微信云调用 或 腾讯云AI | 拍照自动识别水果品类，减轻老板输入 |

> **核心理念**：老板端 → 拍照即上架，顾客端 → 扫一眼就心动。

---

## 二、系统架构与模块划分

采用**单一小程序 + 双端视图**架构，根据登录身份（摊主/顾客）展示不同界面。云端统一管理数据。

```text
┌───────────────────────────────────────┐
│          微信小程序（客户端）           │
│                                       │
│   ┌──────────┐       ┌──────────┐     │
│   │ 老板端    │       │ 顾客端    │     │
│   │ 上架/管理│       │ 浏览/导航│     │
│   └─────┬────┘       └────┬─────┘     │
│         │                 │           │
└─────────┼─────────────────┼───────────┘
          │    HTTPS/云调用   │
          ▼                 ▼
┌──────────────────────────────────────┐
│       微信云开发后端服务              │
│                                      │
│  ┌──────────┐  ┌──────────┐         │
│  │ 云函数    │  │ 云数据库  │         │
│  │ (业务逻辑)│  │ (水果/用户│         │
│  │          │  │ /摊位数据)│         │
│  └──────────┘  └──────────┘         │
│  ┌──────────┐                       │
│  │ 云存储    │                       │
│  │ (水果照片)│                       │
│  └──────────┘                       │
└──────────────────────────────────────┘
```

### 主要功能模块清单

#### 1. 全局通用模块
- **身份识别与切换**：静默登录，通过 `openid` 判断角色，首次引导摊主激活。
- **地理位置服务**：获取用户位置，计算附近摊位。

#### 2. 老板端模块
- **极速上架**：拍照 → 识别水果名 → 填写价格/标签 → 发布。
- **摊位管理台**：水果列表（上下架、改价、标库存）、今日数据概览。
- **摊位设置**：编辑摊位名、位置、联系电话，生成摊位专用小程序码。
- **快捷分享**：生成水果清单海报，一键转发到微信群。

#### 3. 顾客端模块
- **水果橱窗**：首页瀑布流、场景化筛选（“今日推荐”“10元以内”等）、水果详情大图。
- **导航到摊位**：地图组件，展示摊位位置，一键唤起微信导航。
- **收藏与足迹**：收藏常去摊位，查看浏览记录。
- **询问老板**：直接拨打摊主电话（点击拨号）。

## 四、云函数设计

### 1. `fruitManage`（老板端水果操作）
- `addFruit(data)`：接收 name, price, images, tags 等，写入 `fruits` 集合。
- `updateFruit(id, data)`：改价、改库存状态。
- `deleteFruit(id)`：下架（软删除，status 改为 deleted）。

### 2. `fruitList`（顾客端查询）
- `getRecommendedFruits(vendorId?)`：返回 status 为 available/limited 的水果，按时间排序。
- `getFruitsByTag(tag)`：按标签筛选。
- `incrViewCount(fruitId)`：浏览量 +1。

### 3. `vendorInfo`
- `getVendorDetail(vendorId)`：返回摊位信息和今日水果数。
- `updateVendorInfo(data)`：编辑摊位信息。

### 4. `imageAI`（可选）
- `identifyFruit(fileID)`：调用腾讯云图像识别，返回可能的水果名称，供老板确认。

### 5. `share`
- `createShareImage(vendorId)`：云端合成一张水果清单海报（Canvas），返回图片 fileID。

> 云函数使用 `wx-server-sdk`，可以在函数内操作数据库和存储，调用第三方 API。

## 五、关键功能实现要点

### 1. 拍照即上架（老板端核心流程）

**前端**：
```javascript
// 1. 选择或拍照
wx.chooseImage({
  count: 3,
  sizeType: ['compressed'],
  sourceType: ['camera', 'album'],
  success(res) {
    const tempFilePaths = res.tempFilePaths;
    // 2. 上传到云存储，获取 fileID
    uploadImages(tempFilePaths).then(fileIDs => {
      // 3. (可选) 调用云函数识别水果
      wx.cloud.callFunction({
        name: 'imageAI',
        data: { fileID: fileIDs[0] }
      }).then(aiRes => {
        // 返回候选水果名，展示给老板确认
        showConfirmDialog(aiRes.result.name);
      });
    });
  }
});
```

**提交上架**：
```javascript
wx.cloud.callFunction({
  name: 'fruitManage',
  data: {
    action: 'addFruit',
    data: {
      name: '芒果',
      price: 8.8,
      unit: '元/斤',
      images: fileIDs,
      tags: ['新鲜到货'],
      description: '甜过初恋',
      vendorId: getCurrentVendorId()
    }
  }
}).then(res => {
  // 刷新列表，自动回到管理台
});
```

### 2. 首页水果瀑布流（顾客端）
- 使用 `scroll-view` 配合 `onReachBottom` 触底加载。
- 卡片布局：大图 + 价格突出 + 导航小按钮。
- 每次加载 12 条，降低首屏压力。

**瀑布流样式骨架 WXML**：
```xml
<scroll-view scroll-y bindscrolltolower="loadMore">
  <view class="fruit-grid">
    <view class="fruit-card" wx:for="{{fruits}}" wx:key="_id">
      <image src="{{item.images[0]}}" mode="aspectFill" />
      <view class="info">
        <text class="name">{{item.name}}</text>
        <text class="price">¥{{item.price}}/斤</text>
        <view class="status-dot {{item.status}}"></view>
        <button size="mini" bindtap="navigateToVendor">导航</button>
      </view>
    </view>
  </view>
</scroll-view>
```

### 3. 导航到摊位
直接使用微信内置的地图组件，点击后打开第三方导航：
```javascript
wx.openLocation({
  latitude: vendor.location.latitude,
  longitude: vendor.location.longitude,
  name: vendor.stallName,
  address: vendor.address,
  scale: 18
});
```

### 4. 库存状态快速切换
老板端列表每个水果项左滑显示“标为少量/售罄”等快捷操作，调用云函数更新 status。前端通过状态样式（绿、黄、红）直观反映。

### 5. 生成分享海报
云函数用 `node-canvas` 绘制水果清单图片（需使用云端 canvas，微信云开发支持 `canvas` 但不完整，可通过前端 Canvas 生成）。简化方案：前端生成带水印的水果列表长图，保存到相册后分享。

## 六、安全与权限

- 所有数据库操作必须通过云函数，不允许前端直接写库（云开发可设置安全规则）。
- 云函数中使用 `openid` 作为用户唯一标识，老板只能修改自己摊位的水果。
- 上传的图片使用云存储，设置仅用户自己可读（通过 `cloud.uploadFile` 会自动关联 openid）。
- 敏感信息如电话号码可加密存储或仅返回给授权顾客。

## 七、部署与上线步骤简概

1. 注册微信小程序账号（个人主体即可，但部分能力如地图需企业主体，个人可用“打开地图选择位置”代替导航）。
2. 在微信开发者工具新建项目，选择“微信云开发”模板。
3. 开通云开发环境，创建上述数据库集合和云函数。
4. 编写前端界面和逻辑，对接云函数。
5. 添加摊主激活入口（如扫描特定二维码设定为摊主，写入 `vendors` 集合）。
6. 测试，上传代码，提交审核。
7. 上线后，摊主可打印自己的摊位小程序码贴在摊位上。
