# 水果小程序需求文档

## 一、项目概述

**项目名称**：水果小程序
**技术栈**：微信小程序 + 云开发
**目标用户**：商家（发布水果、管理店铺） + 普通用户（浏览购买）
**文档版本**：v2.0
**最后更新**：2026-05-28

## 三、角色与权限

| 角色 | 说明 |
|------|------|
| 店主 (creator) | 店铺负责人，拥有所有权限（邀请/移除管理员、店铺设置） |
| 管理员 (manager) | 被店主邀请的管理者，拥有商品管理、分类管理、公告编辑权限 |
| 普通用户 | 浏览水果、搜索水果、查看详情 |

### 权限矩阵

| 操作 | 店主 | 管理员 | 普通用户 |
|------|------|--------|----------|
| 邀请管理员 | ✅ | ❌ | - |
| 移除管理员 | ✅ | ❌ | - |
| 删除店铺 | ✅ | ❌ | - |
| 店铺设置 | ✅ | ❌ | - |
| 发布/编辑/删除商品 | ✅ | ✅ | - |
| 管理分类 | ✅ | ✅ | - |
| 编辑公告 | ✅ | ✅ | - |
| 上架/下架商品 | ✅ | ✅ | - |
| 浏览水果 | ✅ | ✅ | ✅ |
| 搜索水果 | ✅ | ✅ | ✅ |

## 四、数据模型

### 4.1 店铺表 (shop)

```javascript
{
  _id: ObjectId,
  name: String,              // 店铺名称
  logo: String,              // 店铺Logo
  ownerIds: [String],        // 店主+管理员的openid列表
  creatorId: String,         // 店主openid
  businessStatus: String,    // 营业状态：open（营业）/closed（打烊）
  announcement: String,      // 店铺公告
  createTime: Date
}
```

### 4.2 分类表 (category)

```javascript
{
  _id: ObjectId,
  shopId: String,
  name: String,              // 分类名称，如"榴莲"
  subTitle: String,          // 二级标题（可选）
  description: String,        // 分类描述（可选）
  sort: Number,              // 排序
  createTime: Date
}
```

### 4.3 商品表 (fruit)

```javascript
{
  _id: ObjectId,
  shopId: String,
  name: String,               // 商品名称
  categoryIds: [String],     // 所属分类（可多个，也可为空）
  tags: [String],            // 标签：新上/低价/热卖/预售/售罄
  mainImage: String,          // 主图
  detailImages: [String],    // 详情图（可选）
  description: String,       // 商品描述（可选）
  origin: String,            // 产地（可选）
  status: String,            // 状态：on_sale（上架）/off_sale（下架）
  specs: [{                  // 规格列表
    name: String,            // 规格名，如"大果"
    weight: String,          // 重量，如"2-3斤"
    price: Number,           // 价格（分）
    stock: Number            // 库存
  }],
  createTime: Date,
  updateTime: Date
}
```

### 4.4 邀请表 (invite)

```javascript
{
  _id: ObjectId,
  shopId: String,
  code: String,               // 邀请码
  creatorId: String,         // 创建者openid
  createTime: Date
}
```

## 六、功能详情

### 6.1 用户端功能

#### 首页
- 顶部：店铺名称（硬编码）
- 横向分类滑动条：显示所有分类
- 下方水果卡片列表：图片 + 名称 + 价格（最低价）+ 标签
- 右上角搜索图标
- 点击分类 → 跳转分类页
- 点击水果卡片 → 跳转详情页

#### 分类页
- 显示当前分类名称 + 二级标题（如果有）
- 分类描述（如果有）
- 该分类下的水果列表

#### 详情页
- 商品主图（可放大查看）
- 商品名称
- 标签展示
- 规格选择：用户可查看各规格的重量、价格和库存
- 商品描述
- 产地（如果有）

#### 搜索页
- 顶部搜索框（实时搜索）
- 按水果名称匹配
- 结果展示：图片 + 名称 + 价格 + 所属店铺

### 6.2 商家端功能

#### 商家登录
- 仅授权商家可登录
- 未绑定店铺权限的微信不能进入后台
- 通过邀请链接进入可加入店铺成为管理员

#### 商家首页
- 顶部：店铺名称 + Logo + 营业状态
- 公告展示区域
- 商品筛选标签：全部 / 上架中 / 已下架 / 低库存（≤5件）
- 商品统计：总数、上架数、下架数、低库存数
- 商品列表：图片 + 名称 + 规格数 + 标签 + 状态 + 操作按钮
- 操作按钮：编辑、删除、上架/下架切换
- 快捷入口：上传水果、分类管理、店铺设置、公告编辑、团队管理

#### 发布商品
- 商品名称（必填）
- 选择分类（可多选，可不选）
- 选择标签（可多选）：新上、低价、热卖、预售、售罄
- 上传主图（必填，最多9张）
- 上传详情图（可选）
- 商品描述（可选）
- 产地（可选）
- 规格列表：
  - 添加规格（至少1个）
  - 每个规格：名称 + 重量 + 价格 + 库存

#### 编辑商品
- 同发布页，可修改所有字段
- 支持上下架状态切换

#### 删除商品
- 确认弹窗后删除

#### 上下架管理
- 商品默认上架
- 可随时切换上架/下架状态
- 下架商品不在用户端展示

#### 分类管理
- 添加分类：一级名称 + 二级标题（可选）+ 描述（可选）
- 编辑分类
- 删除分类（有商品时提示）

#### 店铺设置
- 修改店铺名称
- 修改店铺Logo
- 设置营业状态（营业/打烊）

#### 公告编辑
- 发布店铺公告
- 公告显示在商家首页
- 可随时更新公告内容

#### 团队管理（仅店主）
- 当前管理员列表
- 生成邀请链接（复制）
- 移除管理员（店主不可被移除）

## 七、业务规则

| 规则 | 说明 |
|------|------|
| 商家注册 | 店主需先在数据库绑定 openid 和 shopId；管理员通过邀请链接加入 |
| 邀请机制 | 店主生成邀请链接，被邀请人点击即成为管理员 |
| 标签设置 | 预设标签手动选择，可多选 |
| 商品状态 | 商家手动设置上下架状态，默认上架 |
| 分类关联 | 商品可属多个分类，也可无分类 |
| 删除确认 | 删除商品/分类需二次确认 |
| 管理员移除 | 店主可移除管理员，店主自身不可被移除 |
| 低库存预警 | 单品总库存≤5件时显示低库存标识 |
| 价格单位 | 价格为人民币，单位为分，显示时转换为元 |

## 九、技术实现

### 9.1 服务层架构

```
pages/         →  页面逻辑
    ↓
services/      →  业务服务（调用云函数）
    ↓
cloudfunctions/→  云函数（数据操作）
    ↓
database/     →  云数据库
```

### 9.2 数据规范化

使用 `models/fruitMapper.js` 统一处理前后端数据格式：
- `normalizePublicFruit`: 用户端商品数据规范化
- `normalizeMerchantFruit`: 商家端商品数据规范化
- `normalizeFruitDetail`: 商品详情页数据规范化
- `buildMerchantFruitStats`: 商家端商品统计

### 9.3 行为机制

- `authRequired`: 页面访问权限控制，未登录自动跳转登录页
- `refreshOnHomeChanged`: 首页数据刷新控制
